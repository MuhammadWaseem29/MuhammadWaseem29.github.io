XSS to Account Takeover: A Critical Bug in Social Media Platform

September 20, 2025 • Critical Severity • $5000 Bounty

I discovered a stored XSS vulnerability in a popular social media platform that could lead to complete account takeover. This writeup details my findings, exploitation techniques, and the responsible disclosure process.

Discovery

While testing the profile customization feature, I noticed that the platform allowed users to add custom CSS to their profiles. Upon closer inspection, I found that the sanitization logic was incomplete and could be bypassed.

Exploitation

The vulnerability existed in the profile theme feature where users could customize their profile appearance. The CSS validation was insufficient and allowed JavaScript injection via the expression() property in IE and Edge browsers.

body {
  background-color: expression(alert(document.cookie));
}

This code would execute whenever another user viewed the malicious profile, stealing their session cookies and allowing account takeover.

Impact

The vulnerability allowed an attacker to:

  • Steal session cookies of any user viewing the profile
  • Perform actions on behalf of the victim
  • Access private information and messages
  • Potentially escalate to further attacks

Remediation

The company addressed this vulnerability by implementing a more robust CSS sanitization process, including:

  • Removing all potentially dangerous CSS functions
  • Adding a Content Security Policy that blocks inline scripts
  • Implementing proper validation of all user inputs

Timeline

Aug 15, 2025: Vulnerability reported
Aug 16, 2025: Initial response from security team
Aug 20, 2025: Vulnerability confirmed
Sep 05, 2025: Fix deployed
Sep 20, 2025: Bounty awarded ($5000)

Conclusion

This finding highlights the importance of proper input sanitization, especially for features that allow users to inject custom code. Even seemingly harmless features like CSS customization can lead to serious security vulnerabilities if not properly implemented.
